Fix 2 content routes that filter by mode incorrectly for Both users. These are NOT behind mode_isolation_social -- they always filter by mode.

**1. `/api/workshops/route.ts` (lines 79-83):**
The workshops route unconditionally filters by `currentUser.mode`. For Both users, this becomes `WHERE mode = 'both'` which matches nothing since workshops are always 'bible' or 'positivity'.

Add import at top of file: `import { Op } from 'sequelize';` (check if Op is already imported -- if so, skip).

Change from:
```typescript
const currentUser = await User.findByPk(userId, { attributes: ['mode'] });
if (currentUser?.mode) {
  where.mode = currentUser.mode;
}
```
To:
```typescript
const currentUser = await User.findByPk(userId, { attributes: ['mode'] });
if (currentUser?.mode) {
  if (currentUser.mode === 'both') {
    where.mode = { [Op.in]: ['bible', 'positivity'] };
  } else {
    where.mode = currentUser.mode;
  }
}
```

**2. `/api/announcements/active/route.ts` (line 24, 39):**
The announcements route uses `user.mode || 'bible'` then filters `target_mode: { [Op.in]: ['all', userMode] }`. For Both users, this becomes `IN ('all', 'both')` which misses mode-specific announcements targeted at 'bible' or 'positivity'.

Change from:
```typescript
const userMode = user.mode || 'bible';
// ... (later at line 39)
target_mode: { [Op.in]: ['all', userMode] },
```
To:
```typescript
const userMode = user.mode || 'bible';
const targetModes = userMode === 'both'
  ? ['all', 'bible', 'positivity']
  : ['all', userMode];
// ... (update line 39)
target_mode: { [Op.in]: targetModes },
```

**Important:** `Op` is already imported dynamically at line 13 (`const { Op } = await import('sequelize');`) in announcements route, so no additional import needed there. For workshops, check if `Op` is already imported at the top.
