**GET /api/admin/church-outreach/churches:**
- Query params: ?cursor=N&limit=20&stage=...&state=...&search=...
- Filters: pipeline_stage, state (address), free-text search on name/city/pastor_name
- Returns churches with cursor pagination, total count
- Order by created_at DESC

**POST /api/admin/church-outreach/churches:**
- Create a new church manually (not from discovery)
- Zod validation: name required, all other fields optional
- Set source='manual', pipeline_stage='new_lead'
- Create ChurchActivity with activity_type='created'
- Return created church

**GET /api/admin/church-outreach/churches/[id]:**
- Return single church with all fields
- Include latest 10 activities

**PUT /api/admin/church-outreach/churches/[id]:**
- Update church fields (name, pastor_name, contact_email, notes, etc.)
- Zod validation
- If pipeline_stage changed, create ChurchActivity with activity_type='stage_change'
- Return updated church

**DELETE /api/admin/church-outreach/churches/[id]:**
- Hard delete church (cascades to activities, emails, enrollments, shipments, conversion)
- Return 204

**GET /api/admin/church-outreach/churches/[id]/activities:**
- Query params: ?cursor=N&limit=20
- Return activities for this church, ordered by created_at DESC
- Cursor pagination

**PUT /api/admin/church-outreach/pipeline:**
- Body: { churchId: number, stage: PipelineStage }
- Validate stage is a valid PIPELINE_STAGES value
- Update church.pipeline_stage
- Create ChurchActivity with old_stage/new_stage in metadata
- Return updated church

All routes use `withAdmin(handler)` pattern.

**POST /api/admin/church-outreach/discover:**
- Body: { location: string, radiusMiles: number, filters?: string }
- Zod validation: location required (1-100 chars), radiusMiles required (1-50), filters optional
- Calls searchChurches() from google-places module
- For each result, check if google_place_id already exists in churches table (mark as 'already_imported')
- Returns array of PlacesResult objects with already_imported boolean flag
- Returns empty array with message if GOOGLE_PLACES_API_KEY not configured

**POST /api/admin/church-outreach/discover/scrape:**
- Body: { websiteUrl: string, placeId?: string, name: string, address: string }
- Calls scrapeChurchWebsite() from scraper module
- If scrape succeeds, calls researchChurch() from ai-researcher module
- Returns combined result: { scraped: ScrapedData | null, research: ChurchResearchResult | null }
- Returns partial results on scrape or research failure (never throws)

**POST /api/admin/church-outreach/discover/import:**
- Body: { churches: ImportChurchData[] } where ImportChurchData has fields from PlacesResult + optional research data
- Zod validation: churches array, max 100 items
- For each church:
  1. Check for duplicate by google_place_id (skip if exists)
  2. Create Church record with source='google_places', pipeline_stage='new_lead'
  3. If research data provided, populate pastor_name, denomination, youth_programs, etc.
  4. Create ChurchActivity with activity_type='created'
  5. If website was scraped, create ChurchActivity with activity_type='scrape_completed'
  6. If AI research was done, create ChurchActivity with activity_type='ai_researched'
- Returns { imported: number, skipped: number, churches: Church[] }
