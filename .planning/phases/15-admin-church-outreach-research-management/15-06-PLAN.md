**GET /api/admin/church-outreach/sequences:**
- List all drip sequences with step count (subquery or separate query)
- Include enrollment count per sequence
- Order by created_at DESC

**POST /api/admin/church-outreach/sequences:**
- Body: { name, description?, trigger ('manual'|'sample_shipped'|'stage_change'), steps: [{ templateId, delayDays }] }
- Zod validation: name required, steps array required with at least 1 step
- Create DripSequence + DripStep records in a transaction
- Steps created with step_order = array index + 1
- Return sequence with steps

**GET /api/admin/church-outreach/sequences/[id]:**
- Return sequence with all steps (include template name for each step)
- Include active enrollment count

**PUT /api/admin/church-outreach/sequences/[id]:**
- Update sequence name, description, trigger, is_active
- If steps array provided, replace all steps: delete existing, create new (within transaction)
- Return updated sequence with steps

**DELETE /api/admin/church-outreach/sequences/[id]:**
- Check for active enrollments. If any active, return 409 "Sequence has active enrollments"
- Otherwise cascade delete (steps + completed/cancelled enrollments go with it)
- Return 204

**POST /api/admin/church-outreach/sequences/[id]/enroll:**
- Body: { churchId: number } OR { churchIds: number[] }
- For each church:
  1. Check if already enrolled in this sequence (active or paused). If so, skip.
  2. Check if church email is in outreach_unsubscribes. If so, skip.
  3. Create DripEnrollment: status='active', current_step=0, enrolled_at=NOW()
  4. Calculate next_step_at = NOW() + first step's delay_days
- Return { enrolled: number, skipped: number }

**PUT /api/admin/church-outreach/sequences/[id]/enroll:**
- Body: { enrollmentId: number, action: 'pause' | 'resume' | 'cancel' }
- pause: set status='paused'
- resume: set status='active', recalculate next_step_at based on current_step
- cancel: set status='cancelled'
- Return updated enrollment

**src/lib/church-outreach/drip-scheduler.ts:**

Follow the established cron pattern from src/lib/email/scheduler.ts.

**`initDripScheduler(): void`**
- Guard with `globalThis.__dripSchedulerReady`
- Schedule cron: every 15 minutes (`*/15 * * * *`)
- On each tick: call `processPendingDripSteps()`
- Log initialization: `[Drip Scheduler] Cron initialized`

**`processPendingDripSteps(): Promise<void>`**
Implementation:
1. Find all DripEnrollment records where:
   - status = 'active'
   - next_step_at <= NOW()
2. For each enrollment (use SELECT ... FOR UPDATE in transaction to prevent race conditions):
   a. Load the enrollment with sequence and current step
   b. Load the church for this enrollment
   c. Check if church email is in outreach_unsubscribes -- if so, cancel enrollment and skip
   d. If church has no contact_email, skip (don't cancel, they might add one later)
   e. Get the current DripStep (where sequence_id matches and step_order = current_step + 1)
   f. Load the template for this step
   g. Render template with church data using renderTemplate()
   h. Generate UUID tracking_id
   i. Create OutreachEmail record (status='queued', drip_enrollment_id set)
   j. Send via sendOutreachEmail()
   k. Update OutreachEmail status to 'sent'
   l. Create ChurchActivity (activity_type='email_sent', metadata includes drip sequence info)
   m. Update enrollment: increment current_step
   n. Check if there's a next step:
      - If yes: calculate next_step_at = NOW() + next step's delay_days
      - If no: set status='completed', completed_at=NOW()
3. On error for individual enrollment: log error, continue to next (fire-and-forget per enrollment)
4. Log summary: `[Drip Scheduler] Processed N enrollments`

**`enrollInDripSequence(churchId, triggerType): Promise<void>`**
- Helper function for auto-enrollment from triggers (e.g., sample_shipped)
- Find all active sequences with matching trigger type
- Enroll church in each matching sequence (skip if already enrolled)
- Used by sample shipment API to trigger post-sample drip sequences

Export: `initDripScheduler`, `processPendingDripSteps`, `enrollInDripSequence`

Note: The scheduler will be initialized in server.js (wired in plan 15-12 integration).
