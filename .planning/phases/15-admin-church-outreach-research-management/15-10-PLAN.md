**KanbanBoard.tsx:**
- Fetches all churches from GET /api/admin/church-outreach/churches?limit=500 on mount
- Groups churches by pipeline_stage into a Record<string, Church[]>
- Renders 7 KanbanColumn components in a horizontal flex layout

**DnD Setup:**
```typescript
import { DndContext, DragOverlay, closestCorners, PointerSensor, KeyboardSensor, useSensor, useSensors, DragStartEvent, DragEndEvent, DragOverEvent } from '@dnd-kit/core';
```

- Sensors: PointerSensor with activationConstraint { distance: 8 } (prevents click from triggering drag), KeyboardSensor
- Collision detection: closestCorners

**State management:**
- `churches`: Record<PipelineStage, ChurchData[]> -- grouped by stage
- `activeChurch`: the church currently being dragged (for DragOverlay)

**Event handlers:**

`onDragStart(event: DragStartEvent)`:
- Set activeChurch from the dragged item's data

`onDragOver(event: DragOverEvent)`:
- Determine which column the item is hovering over
- Move the church between columns in state (optimistic UI)

`onDragEnd(event: DragEndEvent)`:
- Determine final destination column (stage)
- If stage changed:
  1. Optimistic: already moved in state during onDragOver
  2. Call PUT /api/admin/church-outreach/pipeline with { churchId, stage }
  3. On error: revert the optimistic move, show error toast
- Clear activeChurch

**DragOverlay:**
- Renders a ChurchCard with isDragging=true for the activeChurch
- Provides smooth visual feedback during drag

**Container ID strategy:**
- Each column has id = stage key (e.g., "new_lead", "contacted")
- Each card has id = `church-${church.id}`
- Use the over.id to determine if dropping on a column or another card
- If dropping on a card, determine which column that card is in

**ChurchDetailModal integration:**
- State: selectedChurchId
- On ChurchCard click (not drag): set selectedChurchId, open ChurchDetailModal
- On modal close/update: refresh the churches data
- Import and render ChurchDetailModal from './ChurchDetailModal'

**Layout:**
- Horizontal overflow scroll for the columns
- Full width of admin content area
- Sticky column headers
- Loading skeleton on initial fetch
- Refresh button in top-right corner

**Performance:**
- Limit visible cards per column to 20 with "Show more" button
- Memoize ChurchCard components to prevent unnecessary re-renders during drag
