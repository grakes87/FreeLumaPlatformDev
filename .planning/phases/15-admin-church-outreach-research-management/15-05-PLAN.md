**PUT /api/admin/church-outreach/templates/[id]:**
- Update template fields
- Cannot change is_default flag
- Return updated template

- Check if template is referenced by any campaign or drip step
- If referenced, return 409 "Template is in use"

All routes use `withAdmin(handler)`.

**GET /api/admin/church-outreach/campaigns:**
- List campaigns with cursor pagination
- Include template name (join or separate query)
- Order by created_at DESC

**POST /api/admin/church-outreach/campaigns:**
- Body: { name, templateId, filterCriteria: { stages?: string[], states?: string[], denominations?: string[] } }
- Zod validation
- Creates campaign with status='draft'
- Return created campaign with matching church count (preview how many will receive)

**GET /api/admin/church-outreach/campaigns/[id]:**
- Return campaign with stats (sent_count, open_count, click_count)
- Include list of outreach_emails for this campaign with status

**PUT /api/admin/church-outreach/campaigns/[id]:**
- Update campaign (name, filterCriteria) -- only if status='draft'
- Return 409 if status != 'draft'

**POST /api/admin/church-outreach/campaigns/[id] (action=send):**
- Body: { action: 'send' }
- Only works if status='draft'
- Update status to 'sending'
- Query churches matching filterCriteria:
  - Filter by pipeline_stage if stages provided
  - Filter by state if states provided
  - Filter by denomination if denominations provided
  - EXCLUDE churches with no contact_email
  - EXCLUDE churches in outreach_unsubscribes table
- For each matching church:
  1. Generate UUID tracking_id
  2. Render template with church data (renderTemplate)
  3. Create OutreachEmail record with status='queued', tracking_id
  4. Send email via sendOutreachEmail()
  5. Update OutreachEmail status to 'sent', set sent_at
  6. On error: update status to 'bounced', log error, continue to next church
- After all sent: update campaign status='sent', sent_at=NOW(), sent_count
- Return campaign with final stats

**POST /api/admin/church-outreach/campaigns/[id] (action=cancel):**
- Body: { action: 'cancel' }
- Only works if status='draft' or 'sending'
- Update status to 'cancelled'

Important: The send action processes emails synchronously in the API route for simplicity. If this becomes too slow (>30 seconds), a future optimization would be to use a background queue, but for typical campaign sizes (50-200 churches) this is acceptable.
