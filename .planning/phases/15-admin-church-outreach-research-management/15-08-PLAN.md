This is a PUBLIC endpoint -- NO withAdmin or withAuth wrapper.

**POST /api/sample-request:**
- Body: { churchName, pastorName, email, phone?, addressLine1, addressLine2?, city, state, zipCode, howHeardAboutUs? }
- Zod validation:
  - churchName: string, min 2, max 255
  - pastorName: string, min 2, max 255
  - email: string, email format
  - phone: optional string, max 50
  - addressLine1: string, min 2, max 255
  - city: string, min 2, max 100
  - state: string, min 2, max 50
  - zipCode: string, min 3, max 20
  - howHeardAboutUs: optional string, max 500
  - honeypot: optional string (if filled, silently reject -- anti-spam)

Implementation:
1. Check honeypot field -- if non-empty, return 200 with success message (don't reveal bot detection)
2. Rate limit: 5 submissions per IP per hour (use the existing in-memory rate limiter pattern)
3. Duplicate detection: find existing Church where name LIKE %churchName% AND zip_code = zipCode
   - If found: still return success (don't reveal existing record to public), but DON'T create a duplicate
   - Instead, create a ChurchActivity noting "Duplicate sample request received"
4. If no duplicate: Create Church record with:
   - name: churchName
   - pastor_name: pastorName
   - contact_email: email
   - contact_phone: phone
   - address_line1, address_line2, city, state, zip_code
   - pipeline_stage: 'sample_requested'
   - source: 'sample_request'
   - notes: howHeardAboutUs (if provided)
5. Create ChurchActivity: activity_type='created', description='Submitted sample request via landing page'
6. Send confirmation email via sendConfirmationEmail(email, churchName) -- fire-and-forget (try/catch, don't fail the request)
7. Return 200 { success: true }
