  <action>
All admin routes use `withAdmin(handler)`.

**GET /api/admin/church-outreach/samples:**
- List all sample shipments with church name (join), cursor pagination
- Order by ship_date DESC

**POST /api/admin/church-outreach/samples:**
- Body: { churchId, shipDate (YYYY-MM-DD), trackingNumber?, carrier ('usps'|'ups'|'fedex'|'other'), braceletType?, quantity?, shippingAddress?, notes? }
- Zod validation: churchId required, shipDate required
- Create SampleShipment record
- Auto-advance church pipeline_stage to 'sample_sent' (only if current stage is 'sample_requested' or earlier in the pipeline)
- Create ChurchActivity (activity_type='sample_shipped')
- Call enrollInDripSequence(churchId, 'sample_shipped') to trigger any post-sample drip sequences (fire-and-forget, don't await)
- Return created shipment

**GET /api/admin/church-outreach/conversions:**
- List all conversions with church name, cursor pagination
- Order by order_date DESC

**POST /api/admin/church-outreach/conversions:**
- Body: { churchId, orderDate? (YYYY-MM-DD), estimatedSize?, revenueEstimate?, notes? }
- Zod validation: churchId required
- Create ChurchConversion record (unique per church -- return 409 if already converted)
- Update church pipeline_stage to 'converted'
- Create ChurchActivity (activity_type='converted')
- Return created conversion

**GET /api/admin/church-outreach/reports:**
- Returns aggregated reporting data:
  1. **Pipeline funnel:** Count of churches at each pipeline stage
  2. **Conversion metrics:** Total conversions, conversion rate (converted / total churches), total revenue estimate
  3. **Email metrics:** Total sent, total opened, total clicked, open rate, click rate
  4. **Sample metrics:** Total samples shipped, samples-to-conversion rate
  5. **Activity timeline:** Last 30 days of activities grouped by day (count per day) for CSS bar chart
  6. **Top engaged churches:** Churches with most email opens/clicks (top 10)
- All computed via raw SQL COUNT/SUM queries for efficiency

These are PUBLIC endpoints (no auth). They live at /api/church-outreach/ (NOT /api/admin/).

**GET /api/church-outreach/track?id={trackingId}:**
- Email open tracking pixel endpoint
- Look up OutreachEmail by tracking_id
- If found and not already opened: update status='opened', opened_at=NOW()
- Also update parent OutreachCampaign.open_count (increment by 1) if campaign_id set
- Create ChurchActivity (activity_type='email_opened') -- fire-and-forget
- Return 1x1 transparent GIF (same pattern as existing email tracking in the project)
- Return the GIF regardless of whether the tracking ID was found (don't leak information)
- Use fire-and-forget pattern: don't await the DB updates before returning the GIF

1x1 transparent GIF:
```typescript
const TRANSPARENT_GIF = Buffer.from('R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7', 'base64');
return new Response(TRANSPARENT_GIF, {
  headers: { 'Content-Type': 'image/gif', 'Cache-Control': 'no-cache, no-store' }
});
```

**GET /api/church-outreach/click?id={emailId}&url={encodedUrl}:**
- Click tracking redirect endpoint
- Look up OutreachEmail by id
- If found: update status='clicked', clicked_at=NOW() (only if not already clicked)
- Increment OutreachCampaign.click_count if campaign_id set
- Create ChurchActivity (activity_type='email_clicked', metadata: { url }) -- fire-and-forget
- Redirect (302) to the decoded url
- If url is missing or malformed, redirect to freelumabracelets.com

**GET /api/church-outreach/unsubscribe?email={encodedEmail}:**
- Unsubscribe endpoint
- Decode email from query parameter
- Create or find OutreachUnsubscribe record
- Try to find the church by contact_email to set church_id
- Cancel any active DripEnrollment for this church
- Return a simple HTML page confirming unsubscription: "You've been unsubscribed from Free Luma Bracelets outreach emails."
- The page should be simple, branded, and reassuring
