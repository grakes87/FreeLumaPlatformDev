**src/lib/church-outreach/google-places.ts:**

Create a module for Google Places API (New) Text Search:

1. `geocodeLocation(location: string): Promise<{lat: number, lng: number}>` -- Uses Google Geocoding API to convert ZIP code or city name to lat/lng. Uses env var `GOOGLE_PLACES_API_KEY`. Falls back with error if key not configured.

2. `searchChurches(params: DiscoveryParams): Promise<PlacesResult[]>` where:
```typescript
interface DiscoveryParams {
  location: string;       // ZIP code or city name
  radiusMiles: number;    // radius in miles (converted to meters internally)
  filters?: string;       // optional additional search terms (e.g., "baptist", "youth ministry")
  maxResults?: number;     // default 60, cap at 100
}

interface PlacesResult {
  placeId: string;
  name: string;
  address: string;
  phone: string | null;
  website: string | null;
  types: string[];
  lat: number;
  lng: number;
  googleMapsUrl: string;
  rating: number | null;
  ratingCount: number | null;
}
```

Implementation:
- POST to `https://places.googleapis.com/v1/places:searchText`
- Headers: `X-Goog-Api-Key`, `X-Goog-FieldMask` (only request needed fields to stay in Pro pricing tier)
- Body: textQuery = `churches ${filters || ''}`, locationBias with circle (center lat/lng, radius in meters = miles * 1609.34, max 50000m), pageSize = 20, includedType = 'church'
- Handle pagination: Google returns nextPageToken; loop up to maxResults / 20 pages
- Graceful degradation: if GOOGLE_PLACES_API_KEY not set, throw descriptive error
- Rate limit awareness: add 200ms delay between paginated requests

- Extract social media URLs by matching href patterns for facebook.com, instagram.com, twitter.com/x.com, youtube.com
- Wrap entire function in try/catch, return null on any error
