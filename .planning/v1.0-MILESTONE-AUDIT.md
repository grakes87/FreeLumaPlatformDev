## Integration Gaps

### High Severity

**INT-04: `/api/daily-posts/[date]` broken for Both-mode users**
- File: `src/app/api/daily-posts/[date]/route.ts:37-41`
- `user.mode='both'` passed directly to `WHERE mode = 'both'` → 0 rows → 404
- Also affects `/api/daily-posts/route.ts:26-30`
- **Fix:** Apply `resolveContentMode()` or use ViewModeContext-derived mode
- Requirements: DAILY-01, DAILY-05

### Medium Severity (Latent)

**INT-03: Feed mode-isolation broken for Both-mode users**
- File: `src/app/api/feed/route.ts:68-73`, `src/app/api/feed/fyp/route.ts:91-93`
- `{ mode: 'both' }` WHERE clause matches 0 posts (posts only have 'bible' or 'positivity')
- Currently **latent** — `mode_isolation_social` defaults to `false`
- **Fix:** Use `resolveContentMode()` or expand to `{ mode: { [Op.in]: ['bible', 'positivity'] } }`
- Requirements: FEED-01

### Minor Severity

**INT-01: Daily content bookmarks cannot be created from feed**
- File: `src/hooks/useBookmark.ts:20` — only sends `post_id`
- No bookmark button on `DailyPostSlide`
- API supports `daily_content_id` but no UI path exists
- Requirements: CONT-01, DAILY-07

**INT-02: `meditation_audio_url` not surfaced**
- DB field populated by import scripts but not in API response
- `AudioPlayerSlide` only uses translation-level `audio_url`
- Phase 16 devotional audio partially complete
